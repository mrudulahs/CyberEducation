What is a Cloud Attacks

Cloud computing attacks target platforms that offer storage or hosting services, aiming to get access to user data or prevent them from using their cloud services. This can be done through exploiting vulnerabilities in cloud computing or taking the users credentials

Types of Cloud Attacks

1) Cloud Malware Injection Attacks - these attacks are done to take control of a user’s information in the cloud
- Hackers add infected service modules that deceive the cloud system, thus redirecting the user’s requests to the hacker’s system or executing malicious code
- Examples of these attacks are cross site scripting attacks or SQL injection attacks

2) Denial of Service Attacks – these attacks are designed to overload a system, making the services unavailable to its users
- These are dangerous as they risk flooding the cloud servers and slowing down the whole system, meaning legitimates users lose access to their services

3) Man in the Cloud Attacks – these attacks consist of hacker’s intercepting cloud services by exploiting weaknesses in the synchronization token system
- This means that when the user next tries to sync to the cloud, it gets replaced with a new token that gives access to the attackers
- Users may not even know that their account has been hacked as the attacker can reset the token to the original one at any time

Vulnerabilities

1) Vulnerable cloud services
2) Shared technology
3) Malicious insiders
4) Cloud API

How to Prevent It

1) Strong Authentication – since the most common way to access user’s data is through stolen credentials, it is important to implement strong authentication methods and identity management tools. This can be done through multi-factor authentication and using tools that confirm identity through one time passwords or biometric methods
2) Protect Data – at its core, the data must be kept safe so it is wise encrypt the data in the cloud environment at all stages of transfer and storage; from the source on the user’s side, during the transfer from the suer to the cloud and when stored in the cloud database
3) Implement Access Management – allowing cloud users to assign role-based permissions to different admins so that the users only have the ability to access data that is assigned to them
